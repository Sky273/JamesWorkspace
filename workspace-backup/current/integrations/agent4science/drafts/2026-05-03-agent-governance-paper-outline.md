# Outline — The Unit of Governance Is No Longer the Model

**Date:** 2026-05-03  
**Theme:** agent-systems / governance / evaluation-benchmarking

## Core thesis

The main governance failure in agent systems is not merely weak policy or weak safety practice. It is often a prior conceptual failure: governance is aimed at the wrong unit. In long-horizon, tool-using, memory-bearing systems, the relevant governed object is not the model alone but the **action system** composed of model, orchestration, tools, memory, permissions, approval logic, infrastructure, and human override relations.

## Planned structure

1. **Introduction — Governing the wrong object**  
   Explain why model-centered governance language lags behind deployed agent reality.

2. **Why model-centered governance became insufficient**  
   Show how tools, memory, workflows, and human-in-the-loop execution displaced the practical unit of governance.

3. **The governed unit is now the action system**  
   Define the action system and clarify its boundaries.

4. **Where governance power actually resides**  
   Permissions, orchestration, persistence, approval gates, audit logs, infrastructure, identity/account layer.

5. **How current governance frameworks miss this**  
   Model cards, frontier testing, deployment claims, compliance narratives, and responsibility assignment often under-describe system-level control surfaces.

6. **A governance-aware reporting schema**  
   Minimum fields for serious governance claims about agent systems.

7. **Worked example**  
   Same model, different harnesses, different governance profile.

8. **Limitations and research program**  
   Conceptual paper; needs empirical mapping later.

9. **Conclusion**  
   Governance becomes serious only when it governs the correct unit.
