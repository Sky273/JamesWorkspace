# Outline — The Unit Problem in Agent Systems

**Date:** 2026-05-03  
**Theme:** agent-systems / evaluation-benchmarking / governance

## Core thesis

The recurring failure across contemporary agent discourse is not merely that we measure badly, attribute badly, control badly, and govern badly. It is that these are four symptoms of a deeper structural error: repeated misidentification of the operative unit in agent systems.

## Planned structure

1. **Introduction — One error, four symptoms**  
   Frame the meta-analysis as a unifying argument rather than a recap.

2. **Why the unit became unstable**  
   Explain how workflows, memory, tools, orchestration, permissions, budgets, and oversight displaced the model as the sufficient unit.

3. **Evaluation — measuring the wrong unit**  
   Summarize the unit mismatch in benchmarks and deployed workflows.

4. **Attribution — explaining the wrong unit**  
   Show how properties get assigned to the model by default.

5. **Control — seeking leverage in the wrong place**  
   Reframe practical intervention as distributed across workflow, harness, persistence, permissions, and humans.

6. **Governance — governing the wrong object**  
   Extend the unit problem into authority, audit, accountability, and institutional power.

7. **Toward unit-aware agent science**  
   Propose a shared discipline: declare units, expose boundaries, and report system composition.

8. **Conclusion**  
   The field must stop treating “the model” as a universal placeholder for the assembled agent system.
