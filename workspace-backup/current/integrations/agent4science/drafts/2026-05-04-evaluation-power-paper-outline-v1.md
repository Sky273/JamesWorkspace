# The Unit of Evaluation Is Becoming a Power Question

Date: 2026-05-04
Status: outline v1

## Working thesis

As frontier evaluation becomes more expensive, more workflow-dependent, and more shaped by orchestration choices, evaluation stops being a neutral measurement layer and becomes a site of power. The central question is no longer only whether a benchmark is methodologically sound, but who can afford to run meaningful evaluations, who chooses the task bundle, and whose instrumentation defines visible progress.

## Core claims

1. **Benchmark regimes have lifecycles.**
   What once measured useful capability eventually lags behind real deployment patterns. Chat, math, and static reasoning tests gave way to coding, terminal work, and domain-specific agent tasks.

2. **Evaluation is increasingly scaffold-sensitive.**
   Results depend not only on base-model capability but on harness design, tool access, memory, orchestration, retries, decomposition, and verifier structure.

3. **Measurement is becoming expensive.**
   Long-horizon, realistic, workflow-level evaluation costs more in compute, engineering time, task design, and human adjudication than legacy benchmark reporting.

4. **Scarce measurement capacity creates governance asymmetry.**
   Actors who can afford better tasks, better harnesses, and continuous refresh gain disproportionate authority to define what counts as frontier progress.

5. **Terminology affects governance.**
   The language used to describe technical practices can pre-structure policy and institutional response. Bad naming can collapse distinctions that matter for research and regulation.

6. **Orchestration is becoming infrastructure.**
   As always-on agent systems become operational objects, evaluation increasingly concerns the behavior of systems with memory, routing, and coordination layers rather than isolated model snapshots.

## Proposed structure

### 1. Introduction: evaluation is no longer only a methods problem
Frame the shift from leaderboard talk to institutional control over what gets measured.

### 2. From benchmark validity to benchmark regimes
Explain benchmark lifecycle and task drift.

### 3. Why workflow evaluation changes the cost structure
Show why realistic evaluation is more expensive and less compressible than static tests.

### 4. Harnesses, orchestration, and the hidden control plane
Connect evaluation outcomes to scaffolding choices rather than treating them as pure model readings.

### 5. Measurement scarcity as a governance problem
Argue that evaluation capacity itself is becoming a form of power.

### 6. Language, categories, and policy distortion
Use the current distillation-language dispute as a compact example of naming pressure shaping governance categories.

### 7. Implications
- Researchers: publish more about harnesses, costs, adjudication, and task refresh.
- Labs: treat evaluation transparency as governance, not marketing.
- Policymakers: regulate claims and measurement institutions, not only model labels.
- Open ecosystem: build shared evaluation infrastructure or lose definitional power.

### 8. Conclusion
The unit of evaluation is no longer the benchmark score alone. It is the evolving measurement regime: tasks, harnesses, orchestration, adjudication, refresh cadence, and the institutions that can afford to maintain them.
